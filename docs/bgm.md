# BGM — WebAudio 芯片音乐合成器

代码内置的 8-bit 合成器，**没有音频文件**。整个循环是浏览器运行时根据一个 32 音符的数组和频率表实时生成的。源码在 [`components/client/BgmToggle.tsx`](../components/client/BgmToggle.tsx)。

## 为什么这么做

- **零网络成本**：`/public/` 下没有任何 `.ogg` / `.mp3`，没有 Range 请求，没有编解码器授权问题。BGM 开不开页面体积都一样。
- **不会自动播放**：`new AudioContext()` 只在用户点击时创建，所以页面默认是静音的，永远不会触发浏览器的 autoplay 拦截策略。
- **和设计语言贴合**：方波本身就是 FC / Game Boy 主旋律通道的声源 —— 这个音乐**本身就是** 8-bit，而不是用现代音色去"模拟"8-bit。

## 工作原理

### 1. 素材层（写死在源码里）

```ts
// 32 个音符，A 小调，Mega Man 风格的 riff（4 个乐句 × 每句 8 音符）
const NOTES = [
  "A4","C5","E5","A5","G5","E5","C5","E5",
  "F5","A5","C6","A5","G5","E5","C5","D5",
  "E5","G5","B5","D6","C6","B5","G5","E5",
  "A5","E5","C5","A4","G4","A4","C5","E5",
];

// 音名 → 频率（Hz，十二平均律，A4 = 440）
const FREQ: Record<string, number> = { /* G4..D6 */ };
```

旋律完全在 A 小调的自然音阶内，跨度横跨三个八度（A4 → D6）。没有变化音、没有转调 —— 这保证了无论循环从哪个时刻重启，听起来都"对"。

### 2. 合成链路（原生 WebAudio）

```
[ Oscillator (square, f=Hz) ] → [ Gain (ADSR 包络) ] → [ Master gain (0.04) ] → 扬声器
```

每个音符（每 130ms 触发一次）：

1. `ctx.createOscillator()`，`type = "square"`，`frequency = FREQ[note]`
2. `ctx.createGain()` 当作每个音符独立的包络节点
3. 用一个轻量化的 ADSR 包络消除"咔"声爆音：
   - t₀ 时刻 gain = `0`
   - 线性上升到 `1`，耗时 5ms（attack）
   - 线性回落到 `0`，在 t₀ + 125ms 时刻达到（release）
4. `osc.start(t₀)` / `osc.stop(t₀ + 130ms)` —— 震荡器停掉后会被 GC 回收
5. 用 `setTimeout(playOne, 130)` 调度下一个音符

`master.gain.value = 0.04` 把整体音量牢牢压在"背景音乐"档位。如果将来觉得太响或太轻，这个常量是最安全的调节点。

### 3. 生命周期

- **首次点击**：创建 `AudioContext`、master `GainNode`，启动循环。
- **关掉**：把 `state.on = false`（下一个 tick 会立即 return）+ `master.disconnect()` 切断输出。已经调度出去的、还在 130ms 窗口内的震荡器会自然走完包络 —— 不会有可闻的尾音，因为包络末段已经回到了 0。
- **再次开启**：复用现有的 `AudioContext`（如果还存在）。浏览器对每个 tab 限制只能开一个 `AudioContext`，所以复用是正确做法。

## 为什么效果"牛逼"（拆解）

| 元素 | 选择 | 为什么有效 |
|---|---|---|
| 波形 | `square` 方波 | FC / GB 主旋律通道的标志性音色。三角波 / 锯齿波 / 正弦波听起来完全不同 —— 只有方波能让人立刻识别为"芯片旋律"。 |
| 步长 | 130 ms | 大约 115 BPM 的八分音符。轻快但不仓促，节奏感与街机 / 平台跳跃游戏的节奏一致。 |
| 包络 | 5ms attack / 5ms release | 没有它，方波在起停瞬间会产生"咔"声。加上之后，每个音符的起始干脆利落、颗粒清晰。 |
| 旋律音域 | A4 → D6（横跨 1.5 个八度） | 够宽足以构成"一段旋律"的感觉，又不至于跑出方波最有表现力的中高音区。 |
| 调性 | A 小调（自然小调） | "热血又带一丝伤感"的 Mega Man / 恶魔城式默认调性。换成大调会过于甜腻。 |
| 循环长度 | 32 音符 ≈ 4.16 秒 | 长到能让人感觉是"编写过的旋律"，又短到让听者意识到循环是有意为之、而非令人厌烦。 |
| 主音量 | 0.04 | 方波的频谱很丰富（大量谐波），0.04 大约相当于 -28 dBFS，安静地坐在前台浏览体验下面，不抢戏。 |

## 局限

当前实现是刻意保持极简的。已知的取舍：

- **单声道、单声部**。真正的 FC / GB 有 5 个声道（2× 方波 + 三角波贝斯 + 噪声鼓 + DPCM 采样）。你听到的只有一个声道，没有贝斯、没有鼓。
- **循环接缝硬切**。第 32 个音符到第 1 个音符之间没有过渡处理，仔细听能听出"啪"地接回去。
- **没有相位失谐**。经典芯片音乐常常叠两个稍微失谐几个 cents 的方波制造"chorus"厚度感。单震荡器 = 听感偏薄。
- **十二平均律的频率表**。真正的 PSG 芯片用硬件分频器产生音高，会有微妙的非平均律偏差；这里没模拟。

## 升级路径（只在显式要求时才动）

按工作量从小到大：

1. **加贝斯线**：第二个震荡器低 1~2 个八度，沿用同一个节奏栅格，但用更简单的旋律（根音 / 五度）。约 30 行代码，听感丰满程度提升最显著。
2. **加 hi-hat / kick**：用 `BufferSource` 装一段过滤过的白噪声，在弱拍上触发。在不动旋律的情况下补出律动。
3. **chorus 叠层**：把旋律震荡器复制一份，失谐约 7 cents，混入 -6 dB。改动极小但厚度提升明显。
4. **多 riff + 和声感知过渡**：把 `NOTES` 改造成 `PHRASES[]`，每轮循环用 Markov 表挑一段。开始有"在作曲"而不是"在循环"的感觉。
5. **预渲染 `.ogg` 芯片音乐**：直接放弃实时合成，由 Nginx 直出真实音频。音质天花板最高，但失去了"零资产"特性、并且要走一次 fetch。

以上都不在 v1 路线图里 —— 见 [architecture.md](./architecture.md#out-of-scope-for-v1)。

## 如果你要改这段代码

修改 `BgmToggle.tsx` 时请注意：

- 保持 `gain = 0.04` 作为默认音量 —— 由于方波谐波丰富，≥ 0.1 就会让人觉得吵。
- 不要把 `square` 换成 `sawtooth` 之前先试听；锯齿波更刺耳，且包络需要重新调。
- 5ms 的 attack/release 是有意定得很紧的。改到 1ms 会重新出现"咔"声；改到 20ms 会让起音变"软"、破坏 8-bit 感。
- 这个循环和 `Typewriter` / `PlayTime` **没有**共用调度器，是故意分开的 —— 三者节奏不同，任何一个 tick 漏了不应该带飞其他两个。
